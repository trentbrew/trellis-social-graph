# Plan: Image Posting

## Goal
Allow users to attach an image to a post. Images upload via the Trellis blob store (`client.upload()`), stored by content-addressed hash, and displayed inline in the feed.

## Changes

### 1. `app/lib/schemas/social.ts` — Add `imageUrl` to Post
- Add optional `imageUrl: z.string().optional()` to the Post type attributes
- This stores the blob SHA-256 hash (not a full URL)

### 2. `app/components/ComposePost.client.vue` — Upload UI + image attach
- Add `import { TrellisDb } from 'trellis/client/sdk'` (or use existing `useTrellis()`)
- Add file input (`<input type="file" accept="image/*">`) styled as a camera/image button
- Add `selectedFile` ref to hold the chosen `File` object
- Add image preview strip above the textarea when a file is selected (with an X to remove)
- On submit: if `selectedFile` exists, call `client.upload(await selectedFile.arrayBuffer(), selectedFile.type)` → get `{ hash }`, then include `imageUrl: hash` in the POST body
- 5 MB client-side guard before upload
- Remove unused imports (`useMutation`, `useTrellis` is already imported from the plugin... actually ComposePost uses raw fetch, not the SDK — need to add the SDK client for upload)

### 3. `app/components/SocialFeed.client.vue` — Display images
- When `entry.item.imageUrl` is truthy, render an `<img>` below the post content
- Image URL: `${trellisUrl}/files/${entry.item.imageUrl}` (using runtimeConfig)
- Rounded corners, max-height constraint, object-cover for consistent layout

### 4. `app/components/ComposePost.client.vue` — Switch from raw fetch to SDK
- Currently uses raw `fetch` to `POST /entities` — switch `ensureEntity` and post creation to use `useTrellis()` client for consistency, since we need the client for `upload()` anyway

## Files Modified
- `app/lib/schemas/social.ts`
- `app/components/ComposePost.client.vue`
- `app/components/SocialFeed.client.vue`
